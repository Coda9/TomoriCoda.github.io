关系数据库

关系数据库是支持关系模型的的数据库系统。

关系模型

三要素：

  1. 关系数据结构
  2. 关系操作集合
  3. 关系完整性约束

关系数据结构

域

如D1={李立，王平，刘伟}，m=3;
域的基数表示为m

笛卡尔积

给定一组域D1,D2,…,Dn，这些域可以相同，D1,D2,…,Dn的笛卡尔积为：D1×D2×…×Dn=｛(d1,d2,…,dn)｜di属于Di,i=1,2,…,n｝

即域的叉积
可看作一个二维表

  • 元组
    笛卡尔积中每一个元素(d1,d2,…,dn)
  • 分量
    元素(d1,d2,…,dn)中的每个值di
  • 基数
    D1×D2×…×Dn的基数为 m1×m2×…×mn

关系

笛卡尔积D1×D2×…×Dn的任意子集称为域D1,D2,…,Dn上的关系，表示为R(D1,D2,…,Dn)，其中R为关系名，n是关系的目或度(Degree)。
当n=1时为单元关系，n=2时为二元关系。关系中的每个元素是关系中的元组，通常用t表示。

列：属性
行：元组

特征：

  1. 列来自同一个域
  2. 行与行、列与列之间可交换
  3. 属性不可再分

关系模型

对关系的描述。
关系模型是型，关系是值
可表示为R(U)
R为关系名 U为属性集合

关系完整性

SQL语言

概述

SQL(Structured Query Language)结构化查询语言

具有四种功能：

  1. 数据查询
  2. 数据定义
  3. 数据操纵
  4. 数据控制

涉及到的概念：

  • 基本表
    一个关系对应一个基本表
    一个或多个基本表对应一个存储文件
  • 视图
    从一个或几个基本表导出的表，是一个虚表，即：里面只有定义，没有具体的数据。

SQL支持关系数据库的三级模式
外模式 对应 基本表和视图 ？
模式 对应 基本表
内模式 对应 存储文件
